# 동적 계획법(Dynamic Programming, DP)

- **핵심 아이디어:** 큰 문제를 작은 부분 문제로 나누고, **중복 계산을 저장**해 재사용한다.
- **현업 가치:** 최적화 문제, 비용/경로/스케줄링처럼 **“최소·최대·가능 여부”**를 빠르게 판단할 때 유용하다.
- **주의점:** 부분 문제의 결과가 다시 쓰이는지(중복 부분 문제), 이전 선택이 현재 선택에 영향을 주는지(최적 부분 구조)를 먼저 확인해야 한다.

## 개념 설명

동적 계획법은 한 번 구한 부분 문제의 답을 저장해 두고, 같은 계산을 반복하지 않는 기법이다. 보통 **메모이제이션(top-down)** 또는 **테이블 채우기(bottom-up)** 방식으로 구현한다.  
현업에서는 예를 들어 다음과 같이 자주 등장한다.

- **추천/광고 비용 최적화:** 예산 안에서 캠페인 조합을 선택해 최대 효율을 찾는 문제
- **물류/배송 경로 계획:** 여러 구간의 비용을 누적해 최소 비용 경로를 계산
- **재고·배치 스케줄링:** 제한된 자원으로 최대 처리량을 만드는 의사결정
- **텍스트/문자열 처리:** 편집 거리, 유사도 계산, 시퀀스 비교

DP가 잘 맞는 문제는 보통 상태를 정의할 수 있어야 한다.  
예: `dp[i] = i번째까지 처리했을 때의 최적값`처럼 **상태(state)** 를 정한 뒤, 이전 상태들로 현재 값을 만든다.

## 짧은 예시 / 비유

비유하자면, 등산할 때 매번 같은 경로를 다시 오르는 대신 **중간 쉼터의 기록**을 남겨 다음에 재사용하는 것과 같다.  
예를 들어 `계단 오르기`에서 `dp[i] = i번째 계단까지 가는 방법 수`로 두면, `dp[i] = dp[i-1] + dp[i-2]`처럼 이전 결과를 더해 빠르게 계산할 수 있다.

## 면접 질문 2개

**Q1. DP와 재귀의 차이는?**  
A1. 재귀는 문제를 쪼개는 방식이고, DP는 그 결과를 저장해 **중복 계산을 제거**한다.

**Q2. DP 문제를 처음 만났을 때 무엇부터 보나?**  
A2. `상태 정의 → 점화식 → 초기값 → 계산 순서`를 먼저 정한다. 이 4단계가 보이면 구현이 쉬워진다.

**한 줄 정리:**  
동적 계획법은 **중복되는 최적화 문제를 저장과 재사용으로 빠르게 푸는 핵심 기법**이다.
