# 이진 탐색(Binary Search) 면접 핵심 정리

- **정렬된 배열에서만** 사용할 수 있는 탐색 기법이다.
- 매 단계마다 **탐색 범위를 절반씩 줄여** 시간복잡도는 `O(log N)`이다.
- 구현 실수는 보통 **범위 갱신(left/right)과 mid 계산**에서 발생한다.

## 개념 설명

이진 탐색은 정렬된 데이터에서 원하는 값을 찾을 때, 중간값과 비교해 탐색 범위를 계속 절반으로 줄여 나가는 방법이다.  
예를 들어 `left = 0`, `right = n-1`일 때 `mid = (left + right) / 2`를 계산하고, 찾는 값이 mid보다 크면 왼쪽 절반을 버리고, 작으면 오른쪽 절반을 버린다.

면접에서 자주 묻는 포인트는 다음이다.

- **왜 정렬이 필요한가?**  
  중간값과의 비교 결과로 “왼쪽/오른쪽 중 어느 쪽을 버릴지” 결정하려면 순서 정보가 있어야 한다.
- **시간복잡도는 왜 `O(log N)`인가?**  
  매 단계마다 탐색 구간이 절반이 되므로, 최대 단계 수가 로그에 비례한다.
- **실수하기 쉬운 부분은?**  
  `left <= right`와 `left < right`의 차이, `mid` 갱신 후 무한 루프, 경계값 처리이다.

## 짧은 예시 / 비유

전화번호부에서 이름을 찾는다고 생각하면 된다.  
처음부터 한 줄씩 읽는 대신, 가운데 페이지를 펴서 찾는 이름이 앞쪽인지 뒤쪽인지 판단하고 계속 반으로 좁힌다.  
예: `[1, 3, 5, 7, 9, 11, 13]`에서 `9`를 찾으면 가운데 `7`과 비교 후 오른쪽 절반만 남기고 다시 탐색한다.

## 면접 질문 2개와 짧은 답변

**Q1. 이진 탐색의 조건은 무엇인가요?**  
A. 데이터가 **정렬되어 있어야** 하고, 비교 결과로 탐색 범위를 절반으로 줄일 수 있어야 합니다.

**Q2. `mid = (left + right) / 2`는 항상 안전한가요?**  
A. 언어에 따라 **정수 오버플로우** 위험이 있습니다. 보통 `mid = left + (right - left) / 2`로 작성합니다.

## 한 줄 정리

**이진 탐색은 “정렬된 데이터에서 절반씩 버리며 찾는 `O(log N)` 탐색”이며, 면접에서는 경계 조건 설명이 핵심이다.**
