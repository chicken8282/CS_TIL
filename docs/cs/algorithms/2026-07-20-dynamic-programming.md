# 동적 계획법(Dynamic Programming)

- **핵심 조건:** 문제를 작은 부분 문제로 나눌 수 있고, 부분 문제의 최적해가 전체 최적해를 구성해야 한다.
- **구현 방식:** 재귀 호출 결과를 저장하는 메모이제이션(top-down) 또는 작은 상태부터 배열을 채우는 타뷸레이션(bottom-up)을 사용한다.
- **성능 관점:** 시간 복잡도는 보통 `상태 수 × 전이 비용`, 공간 복잡도는 저장하는 상태 수로 결정된다.

## 개념 설명

동적 계획법은 같은 부분 문제를 반복 계산하지 않도록 결과를 저장하는 기법이다. 먼저 문제를 **상태(state)** 로 표현해야 한다. 예를 들어 배낭 문제에서 `dp[w]`를 “허용 무게가 `w`일 때 얻을 수 있는 최대 가치”로 정의할 수 있다. 상태 정의가 모호하면 점화식과 초기값도 올바르게 만들기 어렵다.

각 상태는 이전 상태에서 **전이(transition)** 된다. 0/1 배낭 문제에서 물건의 무게가 `weight`, 가치가 `value`라면 다음 점화식을 사용한다.

`dp[w] = max(dp[w], dp[w - weight] + value)`

여기서 같은 물건을 한 번만 사용해야 하므로 무게 `w`를 큰 방향으로 순회한다. 반대로 무한히 사용할 수 있는 완전 배낭은 작은 방향으로 순회한다. 즉, **반복문의 방향이 상태 의존성을 통제**한다.

내부적으로는 배열이 가장 흔한 저장 구조다. 2차원 상태를 1차원으로 압축하면 메모리를 줄일 수 있지만, 이전 단계와 현재 단계의 값이 섞이지 않도록 순회 순서를 검증해야 한다. 초기값도 중요하다. 도달 불가능한 상태는 `0`이 아니라 음의 무한대처럼 별도로 표시해야 할 수 있다.

메모이제이션은 필요한 상태만 계산하고 재귀 구조를 그대로 유지하기 쉽다. 그러나 함수 호출 스택과 호출 오버헤드가 생긴다. 타뷸레이션은 반복문 기반이라 스택 오버플로가 없고 캐시 친화적이지만, 상태 계산 순서를 직접 설계해야 한다. 면접에서는 상태 정의, 점화식, 초기값, 순회 방향, 복잡도를 함께 설명하는 것이 핵심이다.

```python
def knapsack(weights, values, capacity):
    dp = [0] * (capacity + 1)
    for weight, value in zip(weights, values):
        for w in range(capacity, weight - 1, -1):
            dp[w] = max(dp[w], dp[w - weight] + value)
    return dp[capacity]

print(knapsack([2, 3, 4], [3, 4, 5], 5))  # 7
```

## 면접 질문

### 1. 동적 계획법과 분할 정복의 차이는?

동적 계획법은 부분 문제가 **중복**되고 결과를 저장한다. 분할 정복은 부분 문제가 보통 서로 독립적이므로 저장이 필수는 아니다.

### 2. 0/1 배낭에서 무게를 내림차순으로 순회하는 이유는?

현재 물건으로 갱신된 `dp[w - weight]`를 같은 반복에서 다시 사용하면 물건을 여러 번 선택하게 된다. 내림차순 순회는 이전 물건 단계의 값만 참조하게 만든다.

## 한 줄 정리

동적 계획법은 상태와 의존 관계를 정의하고, 중복 계산을 저장과 올바른 순회 순서로 제거하는 최적화 기법이다.
