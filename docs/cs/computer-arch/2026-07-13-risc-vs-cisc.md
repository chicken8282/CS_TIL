# RISC vs CISC

- **RISC**는 단순한 명령어를 자주, 빠르게 실행하는 설계다.
- **CISC**는 한 명령어에 더 많은 일을 넣어 코드 길이를 줄이는 설계다.
- 현대 CPU는 둘의 장점을 섞어 **겉은 CISC, 속은 RISC-like**하게 동작하는 경우가 많다.

## 개념 설명

RISC(Reduced Instruction Set Computer)는 명령어 종류를 줄이고, 각 명령어를 단순하게 만들어 **파이프라인 처리**와 **고클럭 설계**에 유리하다. 대표적으로 ARM 계열이 이 철학에 가깝다. 반면 CISC(Complex Instruction Set Computer)는 하나의 명령어가 메모리 접근, 산술 연산 등을 한 번에 처리할 수 있어 **어셈블리 코드 길이**를 줄이는 데 유리하다. 대표 예시는 x86이다.

실무 관점에서 중요한 점은, 현재 x86 CPU도 내부적으로는 복잡한 명령어를 **마이크로 연산(uop)** 으로 분해해 처리한다는 것이다. 즉, ISA는 CISC여도 실행 엔진은 단순 연산 위주로 동작할 수 있다. 그래서 “RISC가 무조건 빠르다” 또는 “CISC가 무조건 느리다”는 식의 단순 비교는 틀리다. 성능은 명령어 수, 분기 예측, 캐시 효율, 메모리 대역폭에 함께 좌우된다.

### 코드 예제
```asm
; CISC 스타일(x86 느낌): 한 번에 메모리에서 더해 저장
ADD DWORD PTR [A], 5

; RISC 스타일(개념적 분해):
LOAD R1, [A]
ADD  R1, R1, 5
STORE [A], R1

; 같은 의미지만 RISC는 단계를 명시적으로 나눈다
```

## 면접 질문

1. **RISC와 CISC의 가장 큰 차이는?**  
   명령어의 복잡도와 실행 방식이다. RISC는 단순 명령어 중심, CISC는 복합 명령어 중심이다.

2. **현대 x86이 CISC인데도 빠른 이유는?**  
   내부적으로 마이크로 연산으로 분해하고, 파이프라인·캐시·분기 예측 최적화가 잘 되어 있기 때문이다.

**한 줄 요약:** RISC/CISC는 “명령어 설계 철학”의 차이이며, 실제 성능은 CPU 내부 구현과 메모리 계층이 더 크게 좌우한다.
