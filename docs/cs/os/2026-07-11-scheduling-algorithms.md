# 스케줄링 알고리즘

- **목적**: CPU 사용률, 처리량, 응답시간, 대기시간의 균형을 맞추는 것
- **핵심 포인트**: 선점형/비선점형, 공정성, 우선순위, 기아(starvation) 발생 여부
- **면접 포인트**: 각 알고리즘의 장단점과 `어떤 상황에 적합한지` 설명할 수 있어야 함

## Concept explanation
스케줄링 알고리즘은 운영체제가 여러 프로세스 중 **어떤 작업에 CPU를 먼저 줄지 결정하는 규칙**이다. 대표적으로 **FCFS(First Come First Served)**, **SJF(Shortest Job First)**, **Priority Scheduling**, **Round Robin**이 있다.

- **FCFS**: 먼저 온 작업부터 처리. 구현이 단순하지만, 긴 작업이 앞에 오면 뒤 작업들이 오래 기다리는 **Convoy effect**가 생긴다.
- **SJF**: 실행 시간이 짧은 작업을 우선. 평균 대기시간이 줄어드는 장점이 있지만, 긴 작업이 계속 밀릴 수 있어 **기아 현상**이 발생할 수 있다.
- **Priority Scheduling**: 우선순위가 높은 작업을 먼저 실행. 중요 작업 처리에 유리하지만, 낮은 우선순위 작업이 계속 밀릴 수 있다.
- **Round Robin**: 각 프로세스에 동일한 시간 할당량(time quantum)을 주고 순환. 인터랙티브 시스템에 적합하며 공정하지만, 문맥 교환 오버헤드가 늘 수 있다.

면접에서는 보통 “**왜 Round Robin이 사용자 응답성이 좋은가?**”, “**SJF가 평균 대기시간이 짧은 이유는?**” 같은 질문이 나온다. 핵심은 **대기시간, 응답시간, 공정성, 오버헤드**의 트레이드오프를 설명하는 것이다.

## Short example or analogy
식당 대기줄로 비유하면:
- **FCFS**: 먼저 온 손님부터 주문
- **SJF**: 빨리 나오는 메뉴부터 처리
- **Priority**: VIP 손님 우선
- **Round Robin**: 모든 손님에게 돌아가며 조금씩 서비스

예를 들어 채팅 앱처럼 빠른 반응이 중요한 서비스는 **Round Robin**이나 유사한 선점형 방식이 적합하다. 반면 배치 작업은 **SJF 계열**이 효율적일 수 있다.

## Interview questions
**Q1. 선점형 스케줄링과 비선점형 스케줄링의 차이는?**  
A. 선점형은 더 높은 우선순위의 작업이 오면 현재 작업을 중단할 수 있고, 비선점형은 CPU를 넘겨줄 때까지 계속 실행한다.

**Q2. Round Robin의 장단점은?**  
A. 장점은 공정하고 응답성이 좋다는 점, 단점은 time quantum이 너무 작으면 문맥 교환 비용이 커지고 너무 크면 FCFS처럼 변한다.

## One-line takeaway
스케줄링은 **“무엇을 먼저 실행할 것인가”**의 문제이며, 면접에서는 각 알고리즘의 **장단점과 적용 상황**을 설명하는 것이 핵심이다.
