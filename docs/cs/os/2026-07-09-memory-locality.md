# 메모리 지역성(Memory Locality)

- **시간 지역성**: 방금 쓴/읽은 데이터는 곧 다시 쓸 가능성이 큼
- **공간 지역성**: 인접한 데이터도 곧 함께 접근할 가능성이 큼
- **성능 핵심**: CPU 캐시를 잘 맞추면 메모리 접근이 빨라짐

## Concept explanation

메모리 지역성은 프로그램이 **데이터를 어떤 순서로 자주 접근하느냐**를 설명하는 개념이다. CPU는 RAM보다 훨씬 빠른 **캐시(Cache)** 를 가지고 있는데, 자주 쓰는 데이터나 그 주변 데이터를 미리 가져와 둔다. 그래서 같은 데이터를 반복해서 읽거나, 인접한 데이터를 연속해서 읽는 코드가 더 빠르다.

지역성은 크게 두 가지다.

- **시간 지역성**: 최근에 접근한 데이터는 곧 다시 접근할 확률이 높다. 예를 들어 반복문 안의 변수, 루프 조건값, 최근 계산 결과가 그렇다.
- **공간 지역성**: 한 위치의 데이터를 읽으면 그 주변 주소의 데이터도 곧 읽을 가능성이 높다. 배열을 순서대로 순회할 때 효과가 크다.

반대로 링크드 리스트처럼 노드가 여기저기 흩어져 있으면 캐시 효율이 떨어질 수 있다. 즉, **CPU가 기다리는 시간을 줄이는 것이 핵심**이다.

## Short example or analogy

비유하자면, 메모리 지역성은 **책상 위 정리 습관**과 비슷하다.

- 자주 보는 펜을 손 닿는 곳에 둔다 → 시간 지역성
- 필요한 노트 주변에 관련 자료를 같이 둔다 → 공간 지역성

예를 들어 배열 `int arr[1000]`을 `for`문으로 앞에서부터 읽으면 캐시가 다음 값도 미리 가져와서 빠르다. 하지만 `arr[0], arr[500], arr[10], arr[700]`처럼 멀리 점프하면 캐시를 잘 못 써서 느려질 수 있다.

## Interview questions

**Q1. 메모리 지역성이 왜 성능에 중요한가요?**  
A. CPU가 RAM보다 훨씬 빠르기 때문에, 캐시 적중률이 높을수록 프로그램이 빨라진다.

**Q2. 배열과 링크드 리스트 중 배열이 캐시에 유리한 이유는?**  
A. 배열은 메모리에 연속 저장되어 공간 지역성이 좋지만, 링크드 리스트는 노드가 흩어져 있어 캐시 효율이 낮다.

**한 줄 요약:** 메모리 지역성은 “가까운 데이터와 최근 데이터부터 잘 활용하면 캐시 성능이 좋아진다”는 원리다.
