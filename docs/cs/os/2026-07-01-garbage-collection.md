# Garbage Collection(GC)

- **역할**: 더 이상 사용되지 않는 객체를 자동으로 찾아 메모리를 회수한다.
- **장점**: 개발자가 직접 `free/delete`를 관리하지 않아도 되어 메모리 누수와 이중 해제 위험이 줄어든다.
- **주의점**: 자동이지만 공짜는 아니다. GC 중단 시간, 힙 사용량, 객체 생성 패턴이 성능에 큰 영향을 준다.

## 개념 설명

Garbage Collection은 프로그램이 더 이상 참조하지 않는 객체를 탐지해 메모리를 반환하는 메커니즘이다. 주로 Java, Go, Python, JavaScript 같은 언어에서 사용된다. 핵심은 **도달 가능성(reachability)** 이다. 루트 스택, 전역 변수, 정적 참조 등에서 시작해 참조 그래프를 따라가며 살아 있는 객체를 구분한다.

대표 방식은 다음과 같다.  
- **Mark-and-Sweep**: 살아 있는 객체를 표시(mark)하고, 표시되지 않은 객체를 청소(sweep)한다.  
- **Generational GC**: 객체는 대부분 금방 죽는다는 가정 아래, 젊은 객체를 자주 수집하고 오래 산 객체는 덜 자주 검사한다.  
- **Copying GC**: 살아 있는 객체만 다른 영역으로 복사해 단편화를 줄인다.

GC의 장점은 생산성이다. 하지만 단점도 명확하다. 큰 힙, 빈번한 객체 생성, 순환 참조, 대량 할당은 GC 부담을 키운다. 특히 실시간성이 중요한 시스템에서는 GC pause가 응답 지연으로 이어질 수 있어 튜닝이 필요하다.

## 짧은 예시 / 비유

비유하자면, 방 정리와 같다.  
책상 위에 있는 물건 중 “지금도 쓰는 것”은 남기고, 더 이상 필요 없는 영수증이나 포장지는 버린다.  
프로그램에서 객체는 물건, 참조는 “사용 중 표시”다. GC는 방 청소를 대신해 주지만, 물건이 너무 많으면 청소 시간이 길어진다.

## 면접 질문 2개

**Q1. GC가 왜 필요한가요?**  
A1. 메모리 해제를 자동화해 누수와 해제 실수를 줄이고, 개발 생산성을 높이기 위해서입니다.

**Q2. GC가 성능에 영향을 주는 이유는?**  
A2. 수집 과정에서 CPU를 사용하고, 경우에 따라 stop-the-world pause가 발생해 지연 시간이 증가할 수 있기 때문입니다.

## 한 줄 요약

GC는 메모리 관리를 자동화하지만, 객체 생명주기와 할당 패턴을 이해해야 성능 문제를 줄일 수 있다.
