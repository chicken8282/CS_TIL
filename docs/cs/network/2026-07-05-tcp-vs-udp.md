# TCP vs UDP

- **TCP**는 연결 지향, **UDP**는 비연결 지향이다.
- **TCP**는 신뢰성·순서 보장, **UDP**는 속도·지연 최소화에 유리하다.
- **TCP**는 웹/파일전송, **UDP**는 실시간 스트리밍/게임에 자주 쓰인다.

## 개념 설명

**TCP(Transmission Control Protocol)**는 통신 전에 연결을 맺고, 데이터가 **정확한 순서로 도착**하도록 관리한다. 패킷 손실이 생기면 재전송하고, 흐름 제어와 혼잡 제어를 통해 네트워크 상황에 맞게 전송량을 조절한다. 그래서 안정적이지만, 그만큼 오버헤드가 있다.

**UDP(User Datagram Protocol)**는 연결 설정 없이 데이터를 바로 보내는 방식이다. 순서 보장, 재전송, 흐름 제어가 기본적으로 없다. 대신 헤더가 단순하고 처리 비용이 적어 **지연이 짧다**. 일부 손실이 있어도 즉시 최신 데이터를 전달하는 게 더 중요한 서비스에 적합하다.

실무에서는 “무조건 TCP가 더 좋다”가 아니라, **정확성 vs 실시간성**의 트레이드오프를 보고 선택한다. 예를 들어 결제, 로그인, 파일 업로드는 TCP가 적합하고, 음성통화, 라이브 방송, 온라인 게임 위치 동기화는 UDP가 자주 사용된다.

## 짧은 예시/비유

TCP는 **택배 배송 + 수령 확인서**에 가깝다. 누락되면 다시 보내고, 순서도 맞춘다.  
UDP는 **확성기로 메시지 전달**하는 느낌이다. 일부 놓칠 수 있지만 빠르게 전달된다.

## 면접 질문

**Q1. TCP와 UDP의 가장 큰 차이는?**  
A. TCP는 연결/순서/신뢰성을 보장하고, UDP는 단순 전송으로 속도를 우선한다.

**Q2. UDP를 쓰는 이유는?**  
A. 지연을 줄이고 오버헤드를 낮추기 위해서다. 실시간성 우선 서비스에 적합하다.

## 한 줄 요약

**TCP는 정확성, UDP는 속도 — 서비스 요구사항에 따라 선택한다.**
