# 프로세스(Process)와 스레드(Thread)

- **프로세스**: 운영체제로부터 **독립된 메모리 공간**을 할당받는 실행 단위
- **스레드**: 프로세스 안에서 **자원을 공유**하며 실행되는 작업 단위
- 핵심 차이: **격리성 vs 경량성**, **안정성 vs 효율성**

## 개념 설명

프로세스는 실행 중인 프로그램으로, 코드/데이터/힙/스택 등 **자기만의 주소 공간**을 가진다. 서로 다른 프로세스는 기본적으로 메모리를 공유하지 않으므로, 한 프로세스가 죽어도 다른 프로세스에 영향이 적다. 대신 생성과 전환 비용이 크고, 프로세스 간 통신(IPC)이 필요하다.

스레드는 프로세스 내부에서 동작하는 실행 흐름이다. 같은 프로세스의 스레드들은 **코드, 데이터, 힙을 공유**하고, 각자 **스택과 레지스터**만 별도로 가진다. 그래서 생성이 빠르고 문맥 전환 비용이 낮지만, 하나의 스레드 오류가 공유 메모리를 망가뜨려 **전체 프로세스 문제**로 번질 수 있다.  
실무에서는 서버 요청 처리, UI 응답성, 병렬 계산 등에 스레드를 활용한다. 반면 충돌 위험 때문에 **동기화(Mutex, Semaphore)** 가 중요하다.

## 짧은 예시/비유

프로세스는 **각자 방이 있는 집**, 스레드는 **같은 방을 나눠 쓰는 가족 구성원**에 가깝다.  
집은 서로 독립적이라 안전하지만 이동이 번거롭고, 가족은 빠르게 협력할 수 있지만 물건을 같이 쓰다 보니 충돌이 생길 수 있다.

예: 브라우저는 탭마다 프로세스를 분리해 안정성을 높이고, 한 탭 안에서는 여러 스레드로 렌더링/네트워크/JS 작업을 병렬 처리할 수 있다.

## 면접 질문

**Q1. 프로세스와 스레드의 가장 큰 차이는?**  
A. 프로세스는 독립 메모리를 갖고, 스레드는 같은 프로세스의 메모리를 공유한다.

**Q2. 왜 스레드가 프로세스보다 가볍다고 하나?**  
A. 스레드는 주소 공간을 새로 만들지 않아 생성/전환 비용이 적고, 자원 공유로 협업이 빠르기 때문이다.

**한 줄 정리:** 프로세스는 **안전한 독립 실행**, 스레드는 **빠른 공유 실행**이다.
