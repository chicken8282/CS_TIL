# 컨테이너 오케스트레이션(Container Orchestration)

- **여러 컨테이너를 자동으로 배치, 실행, 복구, 확장하는 기술**
- **수동 운영의 복잡도를 줄이고 서비스 가용성을 높임**
- **Kubernetes가 가장 널리 쓰이는 대표 도구**

## 개념 설명

컨테이너 오케스트레이션은 여러 컨테이너를 **한 번에 관리하는 운영 체계**입니다.  
컨테이너가 1~2개면 직접 실행해도 괜찮지만, 서비스가 커지면 어떤 서버에 어떤 컨테이너를 띄울지, 장애가 나면 어떻게 다시 살릴지, 트래픽이 늘면 몇 개를 더 늘릴지 직접 처리하기 어렵습니다.

이때 오케스트레이션 도구가 다음을 자동으로 해줍니다.

- **스케줄링**: 컨테이너를 어느 노드에 배치할지 결정
- **자가 복구**: 죽은 컨테이너를 다시 띄움
- **로드 분산**: 여러 인스턴스에 트래픽 분산
- **오토 스케일링**: 부하에 따라 수를 늘리거나 줄임
- **배포 관리**: 무중단 배포, 롤백 지원

쉽게 말해, 컨테이너가 각각의 **배우**라면 오케스트레이션은 **무대감독**입니다. 배우를 어디에 세울지 정하고, 공연 중 문제가 생기면 바로 대체하고, 관객이 몰리면 더 많은 배우를 투입합니다.

## 짧은 예시 또는 비유

배달 앱을 생각해보면 이해가 쉽습니다.  
주문이 적을 때는 배달원 몇 명만 있으면 되지만, 점심 피크 시간에는 더 많은 배달원이 필요합니다. 누군가 갑자기 못 나가게 되면 다른 배달원이 대신 가야 합니다.

컨테이너 오케스트레이션도 비슷합니다.  
웹 서비스 트래픽이 증가하면 Pod를 늘리고, 한 Pod가 죽으면 새 Pod를 띄우고, 여러 Pod에 요청을 나눠줍니다. 사용자는 내부에서 무슨 일이 있는지 몰라도 서비스는 계속 안정적으로 동작합니다.

## 면접 질문 2개

**Q1. 컨테이너 오케스트레이션이 왜 필요한가요?**  
A. 컨테이너가 많아질수록 배포, 복구, 확장, 네트워크 관리가 복잡해지기 때문에 이를 자동화해 운영 효율과 안정성을 높이기 위해 필요합니다.

**Q2. Kubernetes는 오케스트레이션에서 어떤 역할을 하나요?**  
A. 컨테이너를 노드에 배치하고, 장애 시 재시작하며, 서비스 디스커버리와 스케일링, 배포 전략까지 관리하는 대표적인 오케스트레이션 플랫폼입니다.

## 한 줄 요약

컨테이너 오케스트레이션은 **많은 컨테이너를 자동으로 지휘해 서비스 운영을 안정적이고 효율적으로 만드는 기술**입니다.
