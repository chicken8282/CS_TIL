# 컨테이너 오케스트레이션(Container Orchestration)

- **컨테이너가 여러 개일 때** 누가 어디서 실행될지, 얼마나 돌릴지 자동으로 조율하는 기술입니다.
- **배포, 확장, 복구**를 사람이 일일이 하지 않아도 되게 해 줍니다.
- 대표 도구는 **Kubernetes**이며, 실무와 면접에서 거의 기본 용어처럼 자주 나옵니다.

## 개념 설명

컨테이너는 앱을 실행하는 작은 상자라고 볼 수 있습니다.  
문제는 서비스가 커지면 상자가 한두 개가 아니라 수십, 수백 개가 된다는 점입니다. 이때 “어느 서버에 몇 개를 놓을지”, “장애가 나면 어떻게 다시 띄울지”, “트래픽이 늘면 몇 개를 더 만들지”를 사람이 직접 관리하기 어렵습니다.

이런 작업을 자동으로 관리하는 것이 **컨테이너 오케스트레이션**입니다.  
주요 기능은 다음과 같습니다.

- **스케줄링**: 어떤 노드에 컨테이너를 배치할지 결정
- **오토스케일링**: 요청이 늘면 컨테이너 수를 증가
- **자가 치유(Self-healing)**: 죽은 컨테이너를 다시 생성
- **서비스 디스커버리/로드밸런싱**: 여러 컨테이너에 트래픽을 분산
- **롤링 업데이트**: 서비스를 멈추지 않고 새 버전으로 교체

실무에서는 보통 **Kubernetes**가 중심입니다.  
즉, “컨테이너를 실행하는 것”보다 “컨테이너를 운영하는 체계”가 오케스트레이션입니다.

## 짧은 비유/예시

비유하자면, 컨테이너는 **배달 음식 용기**, 오케스트레이션은 **주방의 총괄 매니저**입니다.  
음식이 몇 개 안 되면 사람이 손으로 옮기면 됩니다. 하지만 주문이 폭주하면 누가 어디에 담고, 어떤 배달원에게 주고, 잘못된 주문은 다시 만들지 판단해야 합니다.

예를 들어 쇼핑몰 이벤트 때 트래픽이 급증하면 Kubernetes가 웹 서버 컨테이너를 3개에서 10개로 늘리고, 하나가 죽으면 자동으로 새 컨테이너를 띄워 서비스 중단을 줄입니다.

## 면접 질문 2개

**Q1. 컨테이너 오케스트레이션이 필요한 이유는?**  
A. 컨테이너 수가 많아질수록 배포, 확장, 복구, 운영을 사람이 관리하기 어려워서 자동화가 필요합니다.

**Q2. Kubernetes는 오케스트레이션에서 어떤 역할을 하나요?**  
A. 컨테이너 배치, 스케일링, 장애 복구, 서비스 노출 등을 관리하는 대표 플랫폼입니다.

## 한 줄 요약

**컨테이너 오케스트레이션은 많은 컨테이너를 자동으로 조율해 안정적으로 운영하게 해 주는 기술입니다.**
