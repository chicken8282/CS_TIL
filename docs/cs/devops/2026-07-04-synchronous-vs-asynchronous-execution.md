# 동기(Synchronous) vs 비동기(Asynchronous) 실행

- **동기**는 작업이 끝날 때까지 다음 작업을 기다린다.
- **비동기**는 작업을 요청만 해두고, 완료를 기다리지 않고 다음 일을 진행한다.
- 핵심 차이는 **“누가 기다리느냐”**와 **“처리 흐름을 막느냐”**이다.

## 개념 설명

동기 실행은 호출한 작업이 **순서대로 끝나야 다음 줄로 넘어가는 방식**이다. 예를 들어 함수 A가 끝나기 전까지 함수 B는 시작하지 않는다. 흐름이 단순해서 이해와 디버깅이 쉽지만, 느린 작업이 있으면 전체가 지연된다.

비동기 실행은 작업을 시작한 뒤 **결과가 나올 때까지 현재 흐름을 멈추지 않는다**. 대표적으로 파일 I/O, 네트워크 요청, 타이머, UI 이벤트 처리에서 많이 사용한다. 요청을 던지고 나서 다른 일을 하다가, 완료되면 콜백/Promise/async-await 등으로 결과를 받는다.

중요한 점은 **비동기 = 병렬**이 아니라는 것이다. 비동기는 “기다리지 않고 진행”하는 방식이고, 병렬은 “여러 작업을 실제로 동시에 처리”하는 것이다. 또한 동기/비동기는 실행 흐름의 방식이고, 블로킹/논블로킹은 호출한 스레드가 멈추는지에 대한 관점이다.

## 짧은 예시 / 비유

- **동기 비유**: 식당에서 주문 후 내 음식이 나올 때까지 카운터 앞에서 계속 기다림.
- **비동기 비유**: 주문 후 번호표를 받고 자리로 돌아가 있다가, 진동벨이 울리면 음식 받으러 감.

```js
console.log("1");
setTimeout(() => console.log("2"), 1000);
console.log("3");
// 출력: 1, 3, 2
```

위 예시는 `setTimeout`이 비동기이므로, 1초를 기다리는 동안에도 3이 먼저 출력된다.

## 인터뷰 질문

1. **동기와 비동기의 차이는 무엇인가요?**  
   동기는 이전 작업이 끝나야 다음 작업을 진행하고, 비동기는 완료를 기다리지 않고 다음 작업을 진행합니다.

2. **비동기를 쓰면 항상 성능이 좋아지나요?**  
   항상은 아닙니다. I/O 대기 시간을 숨기는 데 유리하지만, 로직이 복잡해지고 순서 제어가 어려워질 수 있습니다.

**한 줄 요약:** 동기는 순서를 지키며 기다리고, 비동기는 결과를 맡겨두고 다음 일을 먼저 한다.
