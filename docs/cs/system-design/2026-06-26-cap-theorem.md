# CAP 정리(CAP Theorem)

- **일관성(Consistency)**: 모든 클라이언트가 같은 시점에 같은 데이터를 본다.
- **가용성(Availability)**: 모든 요청에 대해 실패 없이 응답한다.
- **분할 허용성(Partition Tolerance)**: 네트워크 단절이 생겨도 시스템이 계속 동작한다.

## 개념 설명

CAP 정리는 **분산 시스템에서 C, A, P를 동시에 완벽히 만족할 수 없다**는 원리다.  
특히 실제 분산 환경에서는 네트워크 장애가 언제든 발생하므로, **P(분할 허용성)는 사실상 필수**로 본다. 그래서 시스템 설계 시 보통 **C와 A 중 무엇을 더 우선할지** 결정한다.

- **CP 시스템**: 데이터 정합성을 우선한다. 네트워크 문제가 생기면 일부 요청을 거부할 수 있다.
  - 예: 금융 이체, 재고 차감
- **AP 시스템**: 응답 가능성을 우선한다. 잠시 데이터가 불일치할 수 있다.
  - 예: SNS 좋아요 수, 추천 피드

중요한 점은 CAP이 “셋 중 두 개만 선택”이라는 단순 공식이 아니라, **네트워크 분할이 발생했을 때 C와 A 중 무엇을 희생할지 정하는 기준**이라는 것이다.

## 짧은 예시/비유

은행 계좌를 생각하면, 두 지점이 네트워크로 끊겼을 때도 잔액이 반드시 같아야 한다면 일부 조회/이체를 막아야 한다. 이것이 **CP**다.  
반대로 쇼핑몰의 상품 추천처럼 잠깐 추천 결과가 다르게 보여도 서비스가 끊기면 안 된다면 **AP**에 가깝다.

## 면접 질문 2개

**Q1. CAP 정리를 한 줄로 설명해보세요.**  
A1. 분산 시스템에서는 네트워크 분할 상황에서 일관성과 가용성을 동시에 완전히 보장할 수 없다.

**Q2. CAP에서 P가 왜 중요하다고 하나요?**  
A2. 분산 시스템은 네트워크 장애를 피할 수 없기 때문에, 실제 설계에서는 P를 전제로 C와 A의 우선순위를 결정해야 한다.

## 한 줄 요약

**CAP 정리는 “분산 시스템은 장애 상황에서 C와 A를 동시에 완벽히 지킬 수 없다”는 설계 기준이다.**
